📋 STEP-BY-STEP SETUP
───────────────────────────────────────────────────────────────────────────────

STEP 1: Add Environment Variables
─────────────────────────────────
Go to v0 Vars (left sidebar) and add:
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY

Get these from Supabase → Project Settings → API


STEP 2: Initialize Database
────────────────────────────
Visit: http://localhost:3000/setup

You'll see TWO OPTIONS:

Option A - Automatic (Easy)
  → Click "Initialize Database" button
  → Wait for success
  
Option B - Manual SQL (Recommended if A fails)
  → Copy the SQL migration
  → Go to Supabase → SQL Editor → New Query
  → Paste SQL
  → Click Run
  → Come back to /setup and click Initialize Database


STEP 3: Start Using codeForge
──────────────────────────────
  ✓ Visit /problems to browse challenges
  ✓ Create account at /auth/signup
  ✓ View dashboard at /dashboard
  ✓ Check leaderboard at /leaderboard

🆘 TROUBLESHOOTING
───────────────────────────────────────────────────────────────────────────────

PROBLEM: "Could not find the table 'public.problems'"
SOLUTION: Go to /setup and use Manual SQL option (Step 2 Option B)

PROBLEM: "Missing Supabase credentials"
SOLUTION: Add env vars in Vars section (Step 1)

PROBLEM: "Failed to initialize database"
SOLUTION: Use Manual SQL option instead of automatic option

Need more help? Check SETUP_INSTRUCTIONS.md
